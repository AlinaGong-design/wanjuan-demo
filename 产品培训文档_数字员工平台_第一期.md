## 二、产品背景与核心概念

### 2.1 什么是数字员工

数字员工是基于大模型构建的、具备特定业务能力的 AI 智能体。它不是通用聊天机器人，而是一个**有身份、有职责、有工具调用能力**的"岗位级 AI"。

- 有固定的**角色定位**（如"法务合规助手"、"HR 招聘助手"）
- 有绑定的**知识库和技能**（如合同规则库、PDF 解析工具）
- 能执行**多步骤、有状态的复杂任务**（不只是回答问题）
- 支持**人工介入节点**（关键决策由人来确认）

## 三、模块一：数字员工使用（前台）

### 3.1 界面总览

前台界面分为三栏：

```
┌──────────────┬─────────────────┬─────────────────────────┐
│   通讯录      │    任务执行状态   │       对话区域            │
│              │                 │                         │
│  · 单员工     │  · 任务列表      │  · 消息流（对话气泡）      │
│  · 群聊       │  · 执行步骤      │  · 工具调用过程展示        │
│  · 置顶       │  · 进度与耗时    │  · 人工确认节点           │
│  · 通知铃铛   │  · 状态筛选      │  · 文件上传区             │
└──────────────┴─────────────────┴─────────────────────────┘
```

### 3.2 通讯录

**功能说明：**

通讯录将单个数字员工和群聊混合展示，按创建顺序排列，不做类型区分。

| 操作 | 说明 |
|------|------|
| 点击条目 | 打开与该员工或群聊的对话 |
| 右键菜单 | 置顶 / 取消置顶；群聊可添加成员、修改群名、修改群头像 |
| 置顶后 | 不论是单员工还是群聊，统一置于列表最上方 |
| 搜索框 | 按名称过滤员工或群聊 |
| 铃铛图标 | 任务完成通知中心，有新完成任务时显示红色角标 |

**销售话术要点：**

> "通讯录和微信的联系人体验一样，员工不需要学习成本。单个 AI 助手和多人协作群聊统一在一个列表里，想和谁协作直接点开就行。"

### 3.3 对话与任务触发

用户通过自然语言与数字员工对话，数字员工会自动将对话意图拆解为可执行的任务步骤。

**对话能力：**

| 能力 | 说明 |
|------|------|
| 自然语言输入 | 直接描述需求，无需填写表单 |
| 文件上传 | 支持上传 PDF、Word 等文档，员工自动解析 |
| @提及 | 在群聊中 @ 特定员工，精准分配任务 |
| DeepThink | 开启深思模式，员工进行多步骤深度推理 |

**任务执行透明度（重要卖点）：**

每个任务的执行过程完全可见：

```
用户发送"帮我审核这份合同"
  ↓
员工回复：已收到，开始处理
  ↓
[步骤 1 ✓] 文档解析  — 耗时 3s — 识别到 18 个条款段落
[步骤 2 ✓] 规则匹配  — 耗时 8s — 命中法规库 203 条
[步骤 3 ✓] 风险识别  — 耗时 10s — 发现 2 处高风险条款
[步骤 4 ⏳] 人工确认 — 等待您确认处置方案
```

> **卖点：** AI 不是黑盒，每一步做了什么、用了什么工具、输出了什么结果，用户全程可见、可追溯。

### 3.4 任务完成通知

通讯录顶部铃铛图标，汇聚所有数字员工的任务完成推送：

- 显示员工名称、任务标题、执行结果、完成时间和耗时
- 未读通知有蓝色小圆点标记
- 支持逐条已读或一键全部已读

**使用场景：** 用户同时给多个数字员工分配了任务，不需要逐个查看对话，在通知中心统一查看所有完成情况。

### 3.5 群聊协作

| 操作 | 说明 |
|------|------|
| 创建群聊 | 在单聊对话头部点击 `+` 按钮，可将单聊升级为群聊 |
| 群聊中添加成员 | 在群聊对话头部点击 `+`，选择要加入的数字员工 |
| 修改群信息 | 点击群聊头像，可修改群名称和群头像 Emoji |
| 右键菜单 | 可从通讯录右键菜单进入群成员管理 |

**典型协作场景：**

> 法务合规助手 + 合同审核助手 + 智能巡检助手 建一个项目群，用户在群里发一个复杂任务，@各自擅长的员工分工处理。

## 四、模块二：员工库空白配置

### 4.1 配置流程总览

空白配置是从零创建一个新的数字员工，分为 **4 个步骤**：

```
第一步          第二步              第三步          第四步
基础信息   →   AI 能力配置   →   部署配置   →   测试发布
（谁）          （能做什么）       （给谁用）      （验证上线）
```

### 4.2 第一步：基础信息

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 员工名称 | 该数字员工的身份标识 | 合同审核助手 |
| 所属部门 | 归属的业务部门 | 法务部 |
| 角色定位 | 一句话描述岗位职责 | 法务合规顾问 |
| 功能描述 | 展示在员工库列表中的简介 | 专注合同风险识别与条款审查 |

### 4.3 第二步：AI 能力配置（核心）

**模型选择：**

| 模型 | 适用场景 | 推荐场景 |
|------|----------|----------|
| chat-model | 通用对�� | 日常 Q&A 型员工 |
| GPT-4o | 复杂推理 | 合同审查、代码分析 |
| Qwen-Max | 长文本处理 | 大文档解析、报告生成 |
| GLM-4 | 国产合规 | 对数据安全有要求的客户 |

**提示词（System Prompt）：**

这是员工的"灵魂"，定义员工的职责边界、工作方式和输出规范。

> **配置原则：**
> 1. 明确角色身份：你是谁、你负责什么
> 2. 限定工作范围：只处理哪类问题，不在范围内如何回应
> 3. 规定输出格式：回复风格、结构化程度
> 4. 注入业务规则：公司特有的流程、规范、优先级

**知识库绑定：**

| 知识库类型 | 典型内容 |
|------------|----------|
| 法律法规库 | 合同法、数据安全法、行业规范 |
| 公司制度手册 | 内部流程、审批规则、行为准则 |
| 产品知识库 | 产品说明书、FAQ、技术文档 |
| 历史案例库 | 过往项目经验、解决方案沉淀 |

**平台技能（工具）：**

| 技能 | 说明 |
|------|------|
| PDF 解析 | 读取 PDF 文档内容 |
| 网络搜索 | 实时查询公开信息 |
| 飞书文档读写 | 读取和写入飞书文档 |
| Python 执行 | 执行数据分析脚本 |
| 工单创建 | 对接工单系统创建任务 |
| 飞书推送 | 发送消息到飞书群 |

### 4.4 第三步：部署配置

| 配置项 | 选项 |
|--------|------|
| 回复风格 | 正式严谨 / 专业友好 / 简洁直接 |
| 访问权限 | 仅创建者 / 部门内 / 全公司 |
| 接入渠道 | 前台嵌入 / REST API / 工作流节点 / 飞书机器人 / 钉钉机器人 |

### 4.5 第四步：测试发布

发布前系统自动做配置完整性校验：

```
✓ 基础身份信息完整
✓ 模型已选择
✓ 提示词长度合规（≥ 50 字）
✓ 至少绑定一个知识库或技能
✓ 访问权限已配置
```

测试通过后点击"发布"，员工立即出现在通讯录可被调用。

### 演示案例一：合同风险审核助手

**适用行业：** 制造业、贸易、科技公司
**对应客户痛点：** 法务人员少、合同量大、审核周期长、风险靠人工经验

**配置信息（现场按以下内容填写）：**

---

**第一步：基础信息**

```
员工名称：合同风险审核助手
所属部门：法务部
角色定位：法务合规顾问
功能描述：专注企业合同风险识别与条款审查，支持 PDF 合同上传，自动提取关键条款，识别违约责任、数据归属、单方解约等高风险项，生成结构化审查报告
```

**第二步：AI 能力配置**

```
模型：GPT-4o（合同审查属于高精度推理任务）

提示词：
你是一位专业的法务合规顾问，负责企业合同的风险审查工作。

你的职责范围：
1. 接受用户上传的合同文档（PDF/Word）
2. 解析合同结构，识别关键条款（甲乙方信息、合同金额、违约责任、数据权属、解约条件等）
3. 对每个关键条款进行合规性评估，输出风险等级（高/中/低）和修改建议
4. 对于高风险条款，主动向用户寻求确认意见（接受修改建议 / 保留原文并备注风险）
5. 最终输出结构化审查报告

工作原则：
- 发现的风险必须给出明确的依据（引用法条或公司合同规范）
- 每条高风险必须给出具体的修改建议，而非仅标注风险
- 审查结论需区分"建议签署前修改"和"可接受但需备注"两个等级
- 不在合同审查范围内的问题（如税务、财务条款解读），引导用户联系对应部门

输出格式：
- 合同基本信息摘要（甲乙方、金额、有效期）
- 风险清单（逐条列出，高风险优先）
- 每条风险：[条款名称] + [风险描述] + [建议处理方式]
- 审查结论与签署建议

知识库：
✓ 法律法规库（合同法、民法典、数据安全法）
✓ 公司合同审查规范手册

技能（工具）：
✓ PDF 解析
✓ 飞书文档读写（输出审查报告到飞书）
```

**第三步：部署配置**

```
回复风格：正式严谨
访问权限：部门内（法务部及各业务部门负责人）
接入渠道：前台嵌入 + 飞书机器人
```

**演示对话脚本：**

> 配置发布后，在前台通讯录点击"合同风险审核助手"，上传一份示例合同 PDF，发送：
>
> `"帮我审核这份供应商框架协议，重点看违约责任和数据权属条款"`

预期展示效果：
- 员工逐步展示文档解析、规则匹配、风险识别的过程
- 对高风险条款触发人工确认交互
- 最终输出结构化审查结论

**销售说明要点：**

> "以前一份合同审查需要法务至少花 1-2 天，用了这个助手，15 分钟内就能得到初审结果。法务人员从"逐字看合同"解放出来，专注在高价值的决策判断上。"

### 演示案例二：招聘简历筛选助手

**适用行业：** 快速扩张期企业、HR 团队偏小的科技公司、校园招聘量大的企业
**对应客户痛点：** 招聘季简历量爆炸、初筛耗费大量 HR 时间、筛选标准不统一

**配置信息（现场按以下内容填写）：**

---

**第一步：基础信息**

```
员工名称：招聘简历筛选助手
所属部门：人力资源部
角色定位：HR 招聘专员
功能描述：支持批量上传简历，自动解析并按岗位要求进行多维度评分与排名，识别重点匹配项和潜在风险，生成候选人对比表，减少 HR 初筛工作量 80%
```

**第二步：AI 能力配置**

```
模型：Qwen-Max（简历批量处理属于大文档场景，长文本处理能力更强）

提示词：
你是一位专业的 HR 招聘专员，负责岗位简历的初步筛选工作。

你的职责范围：
1. 接受用户上传的简历文件（单份或批量，支持 PDF/Word）
2. 接受用户描述的岗位要求（或从 JD 文档中自动提取）
3. 对每份简历进行多维度评分：
   - 技能匹配度（与岗位 JD 的契合程度）
   - 工作年限与经验质量
   - 教育背景
   - 成就量化程度（有无具体数字说明成果）
   - 稳定性（平均在职时长）
4. 输出综合评分（0-100分）和排名
5. 对进入候选的简历，给出优势总结和潜在风险点

工作原则：
- 评分需客观，不受简历格式和措辞的干扰，聚焦实质内容
- 对于信息不完整的简历，明确标注"信息缺失"而非猜测
- 批量处理时，发现格式异常的简历需单独提示，不影响其他简历处理
- 不做歧视性筛选（年龄、性别、地域等不作为评分维度）

输出格式：
- 处理摘要（共处理 N 份，成功 N 份，异常 N 份）
- 候选人排名表（姓名、综合评分、核心优势、关注点）
- 每位候选人详细评分卡（各维度得分 + 依据说明）
- 建议进入面试的候选人名单

知识库：
✓ 岗位 JD 库（公司历史招聘岗位的要求模板）
✓ 人才评估标准手册

技能（工具）：
✓ PDF 解析（读取简历文件）
✓ 飞书多维表格（输出候选人对比表）
✓ 飞书推送（通知 HR 负责人审核）
```

**第三步：部署配置**

```
回复风格：专业友好
访问权限：部门内（HR 部门及各用人部门负责人）
接入渠道：前台嵌入 + 飞书机器人
```

**演示对话脚本：**

> 配置发布后，在前台通讯录点击"招聘简历筛选助手"，发送：
>
> `"我们在招一个高级 Java 工程师，要求 5 年以上经验，熟悉分布式架构，这里有 30 份简历，帮我筛一下"`
>
> （可上传 2-3 份示例简历 PDF 演示）

预期展示效果：
- 员工展示批量解析过程和进度
- 对每份简历进行多维度评分
- 输出候选人排名和对比表
- 推荐进入面试的候选人

**销售说明要点：**

> "校招季一天可能收到几百份简历，HR 光做初筛就能花掉 2-3 天。有了这个助手，30 分钟内完成所有简历的结构化评分，而且评分标准统一，不会因为 HR 个人偏好导致优秀候选人被漏掉。HR 把精力放在面试和 Offer 谈判上，这才是他们最有价值的地方。"

## 六、常见客户问题 & 解答

| 客户问题 | 解答要点 |
|----------|----------|
| "跟普通 ChatGPT 有什么区别？" | 有固定岗位身份、绑定企业私有知识库、能调用内部系统（飞书/工单/数仓），不是通用聊天，是"在岗 AI 员工" |
| "数据安全怎么保障？" | 支持私有化部署，可选择国产模型（GLM-4），企业数据不出域 |
| "员工回答错了怎么办？" | 关键节点设置人工确认，高风险决策由人来拍板；同时执行过程全程透明可追溯 |
| "配置一个员工需要多久？" | 基础配置（填信息+提示词+选技能）30 分钟内完成；深度调优需要 1-2 周磨合 |
| "能接多少个系统？" | 当前支持飞书、钉钉、REST API 接入；后续工作流和更多 MCP 插件持续开放 |
| "现有的 IT 系统需要改造吗？" | 前台使用零改造；API 接入需要 IT 配合，通常 1-3 天完成对接 |
