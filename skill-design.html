🎯 Skill 技能管理模块

智能体群组协作系统 - 核心功能设计文档

一、模块概述

Skill技能管理模块是智能体能力扩展的核心系统。通过插件化的Skill设计， 智能体可以调用外部API、执行特定函数、检索知识库等，极大增强了AI的实用价值。

💡 核心理念
Skill就像智能体的"工具箱"，每个Skill代表一种特定能力。智能体通过组合不同的Skill， 可以完成从数据分析、内容创作到业务集成的各类任务。

二、核心功能

📋 技能列表展示

卡片式网格布局，直观展示所有技能。支持按状态筛选（已发布/草稿/下架）， 实时搜索快速定位目标技能。

✏️ 技能创建与编辑

可视化编辑器支持三种技能类型配置：API调用、函数工具、知识库。 内置JSON Schema编辑器，规范参数定义。

🧪 在线测试功能

编辑器内置测试模块，输入测试数据即可实时验证技能效果。 支持查看请求/响应日志，快速调试。

🔄 状态生命周期管理

草稿 → 已发布 → 下架的完整流程。已发布技能可被智能体关联使用， 下架后不影响已关联智能体。

📊 使用统计

实时统计技能被多少智能体使用，帮助管理员了解热门技能， 优化技能库资源配置。

🔗 智能体关联

在智能体配置页面，通过Drawer侧边栏选择已发布的技能。 支持多选，灵活组合智能体能力。

三、技能类型详解

3.1 API调用类型

允许智能体调用外部HTTP API，实现与第三方服务的集成。 适用场景：天气查询、数据获取、业务系统对接等。

┌──────────────────────────────────────┐ │ API调用配置 │ ├──────────────────────────────────────┤ │ 接口地址: https://api.example.com/v1│ │ 请求方法: [POST ▼] │ │ │ │ 请求参数 (JSON Schema): │ │ { │ │ "type": "object", │ │ "properties": { │ │ "query": { │ │ "type": "string", │ │ "description": "查询关键词" │ │ } │ │ }, │ │ "required": ["query"] │ │ } │ │ │ │ 返回格式: │ │ { │ │ "type": "object", │ │ "properties": { │ │ "result": { "type": "string" } │ │ } │ │ } │ └──────────────────────────────────────┘

3.2 函数工具类型

支持自定义JavaScript/Python函数，处理复杂的业务逻辑。 适用场景：数据计算、格式转换、复杂规则判断等。

function calculateDiscount(orderAmount, userLevel) { // 根据订单金额和用户等级计算折扣 const discountRates = { 'vip': 0.8, 'gold': 0.9, 'normal': 0.95 }; return orderAmount * (discountRates[userLevel] || 1); }

3.3 知识库类型

关联向量化的文档库，支持语义检索。智能体可以从大量文档中快速找到相关信息。 适用场景：企业文档问答、产品手册查询、政策法规检索等。

✅ 技术实现
使用向量嵌入（Embedding）技术将文档转为向量存储，通过余弦相似度检索Top-K相关片段， 结合大语言模型生成最终答案（RAG架构）。

五、业务流程

技能创建与发布流程

1

填写基础信息

输入技能名称、选择图标、设置类别和描述

2

选择技能类型

根据业务需求选择：API调用、函数工具或知识库

3

配置技能参数

API类型：配置接口地址、请求方法、参数Schema
函数类型：编写函数代码
知识库类型：关联文档并配置检索参数

4

添加使用示例

提供输入输出示例，帮助理解技能用途

5

��试验证

输入测试数据，调用技能接口，检查返回结果是否符合预期

6

发布技能

设置状态为"已发布"，技能进入可用库，可被智能体关联

⚠️ 注意事项
• 草稿状态的技能无法被智能体使用
• 已被智能体使用的技能删除时会显示警告
• 下架的技能不会影响已关联的智能体，但不可被新智能体选择

七、状态管理

📝 草稿 (draft)

编辑中的技能，未通过测试或未完成配置。
限制：无法被智能体选择关联。

✅ 已发布 (published)

已完成配置并通过测试的技能。
可用：智能体可以选择并使用此技能。

🗄️ 下架 (archived)

不再推荐使用的技能。
说明：已关联的智能体仍可使用，但不可被新智能体选择。

状态流转规则

草稿 (draft) ↓ │ 配置完成并测试通�� ↓ 已发布 (published) ↓ │ 管理员主动下架 ↓ 下架 (archived) ↓ │ 重新启用 ↓ 已发布 (published)

八、技能调用流程

智能体调用Skill的完整流程

1

用户发送任务请求

用户在聊天界面向智能体提出需求，例如："帮我分析这份销售数据"

2

智能体判断是否需要技能

AI分析任务需求，判断是否需要调用特定技能完成任务

3

查询可用技能列表

从智能体关联的技能列表中，匹配最合适的技能

4

准备调用参数

根据技能的参数Schema，从用户输入中提取所需参数

5

执行技能

• API类型：发送HTTP请求到指定接口
• 函数类型：执行函数代码
• 知识库类型：检索相关文档片段

6

获取技能返回结果

接收技能执行的结果数据

7

AI生成最终回复

将技能返回的数据结合AI的理解能力，生成自然语言回复

8

展示给用户

在聊天界面显示最终的回复内容

十、最佳实践

💡 设计建议

1. 技能粒度控制

✅ 推荐：一个技能完成一个明确的功能（单一职责原则）
❌ 避免：一个技能包含多个不相关的功能，增加维护成本

2. 参数设计规范

• 使用JSON Schema严格定义参数类型
• 为每个参数添加清晰的description
• 合理设置required字段，避免过多必填项

3. 错误处理

• API调用失败时返回友好的错误信息
• 设置合理的超时时间
• 关键技能建议配置重试机制

4. 性能优化

• 对高频调用的API结果进行缓存
• 知识库检索结果设置合理的topK值
• 函数工具避免耗时过长的同步操作

5. 安全性考虑

• API Key等敏感信息使用环境变量存储
• 对外部API的返回数据进行校验
• 函数工具代码执行在沙箱环境中

十一、技术实现要点

前端实现

  • 基于 Ant Design 的Form组件构建编辑器
  • 使用 Monaco Editor 实现JSON Schema编辑
  • 卡片网格布局使用 CSS Grid 实现响应式
  • 状态管理使用 Zustand 或 Redux

后端实现

  • API调用使用 axios 或 fetch
  • 函数工具在 VM2 沙箱环境中执行
  • 知识库使用 Pinecone / Qdrant 向量数据库
  • 技能配置存储在 PostgreSQL / MongoDB
