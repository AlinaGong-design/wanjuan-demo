## 1. 背景与目标

### 1.1 背景

企业 AI 落地面临三大核心痛点：

| 痛点 | 现状描述 |
|------|---------|
| **配置门槛高** | 绑定知识库、调试 Prompt、对接内部系统需较强技术能力，业务人员难以独立完成 |
| **管理碎片化** | AI 能力散落在多个工具，缺乏统一的生命周期管理（版本、权限、监控） |
| **价值难量化** | 缺乏面向管理层的可视化度量体系，AI 投入效益无从衡量 |

数字员工模块旨在提供 **"配置即服务、使用即交付"** 的完整闭环，让非技术业务人员能够独立完成从创建到运行的全流程。

### 1.2 产品目标

**核心命题：** 非技术业务人员能够独立完成数字员工的从零到发布；任意员工能够自然地通过对话使用数字员工完成业务任务。

| 目标维度 | 衡量标准 |
|---------|---------|
| 配置效率 | 非技术人员完成创建 → 配置 → 发布全流程 ≤ 45 分钟 |
| 使用体验 | 前台对话满意度评分均值 ≥ 4.2 星 |
| 数据互通 | 评分数据从前台实时回流至管理台，延迟 ≤ 3s |
| 功能完整性 | P0 功能 100% 交付 |

### 1.3 与主平台的关系

数字员工模块作为万卷企业 AI 平台的核心业务模块，通过 `#digital-employee-*` hash 路由接入主平台导航，数据层通过 `employeeStore` 发布-订阅机制与其他模块共享状态，实现评分、调用量的实时跨页联动。

**③ 普通员工**

| 诉求 | 优先级 |

**④ 数据分析师**

| 诉求 | 优先级 |

## 3. 整体架构

### 3.1 模块地图

```
数字员工模块
├── 配置侧（管理台）
│   ├── 员工库（Library）         # 全生命周期管理
│   │   ├── 员工列表（卡片/列表视图）
│   │   ├── 创建/编辑向导
│   │   ├── 版本管理 Drawer
│   │   └── 多入口接入 Drawer
│   ├── 工作台（Workbench）P1     # 数据大盘
│   ├── 360° 画像（Profile）P1   # 价值量化
│   ├── 定时任务（Scheduled）P1   # 自动化执行
│   └── 业务域（Domain）P2        # 组织治理
│
├── 使用侧（前台）
│   └── 前台对话（Frontend）      # 员工选择 + 多轮对话 + 评分
│
└── 共享层
    └── employeeStore             # 跨模块实时数据同步
```

### 3.2 核心数据流

```
用户在前台发起对话并评分
    ↓
employeeStore.submitRating()
    ↓  notify() 触发所有订阅者刷新
    ├── 员工库：评分数值实时更新
    ├── 工作台：调用量 KPI +1
    └── 360° 画像：用户满意度维度更新
```

## 5. P0 · 数字员工配置侧

### 5.1 员工库（DigitalEmployeeLibrary）

#### 5.1.1 员工列表

**卡片视图（默认，3 列网格）**

每张卡片包含：
- 顶部渐变色条（基于员工名称哈希生成，视觉区分同名员工）
- 员工头像（40×40，渐变色，展示姓名���字）
- 员工名称 + 类型标签（通用款 / 定制款 / 升级款）+ 业务域
- 状态 Badge（草稿灰 / 测试中蓝 / 运行中绿 / 已暂停橙 / 已归档红）
- 三核心指标：调用量（≥1000 转 k 单位）/ 用户评分（⭐ + Tooltip 说明抽样人数）/ 活跃热度（近 7 日活跃率 %）
- 部门 · 版本 · 更新日期
- 行内操作按钮组

**列表视图（表格，用户手动切换）**

列：数字员工（头像+名称+部门·域·版本）/ 类型 / 状态 / 访问权限 / 调用量 / 评分 / 更新时间 / 操作

**筛选与搜索**

| 功能 | 描述 |
|------|------|
| 名称/描述搜索 | 输入即过滤，支持 allowClear |
| 状态筛选下拉 | 与搜索条件 AND 组合 |
| 业务域 Tab | 颜色 Tag 单选过滤，"全部"默认选中 |
| 结果计数 | 工具栏右侧「共 N 个」 |

#### 5.1.2 员工生命周期状态机

```
[草稿 draft] ──→ [测试中 testing] ──→ [运行中 published]
     │                  │                    │      ↑
     │  直接发布          └── 直接发布         ↓      │ 恢复
     │                              [已暂停 paused]   │
     └─────────────────────────────────────────────→ │
                                                      │
                    任意状态 ────────→ [已归档 archived]（不可恢复）
```

| 操作 | 触发条件 | 交互规则 |
|------|---------|---------|
| 发布 | draft / testing | 无强制前置约束，message.success 提示 |
| 暂停 | published | 直接暂停，无需确认 |
| 恢复 | paused | 直接恢复为 published |
| 归档 | 任意非 archived | Modal.confirm 二次确认，OK 按钮 danger 样式 |

#### 5.1.3 行内操作

| 操作 | 显示条件 | 交互 |
|------|---------|------|
| 编辑 | 所有状态 | 打开全页配置向导 |
| 版本 | 所有状态 | 打开版本管理 Drawer（480px） |
| 接入 | 所有状态 | 打开多入口接入 Drawer（520px） |
| 发布 | draft / testing | 直接发布 |
| 暂停 | published | 直接暂停 |
| 恢复 | paused | 直接恢复 |
| 归档 | 非 archived | Modal.confirm |

// 评分提交链路
submitRating(rating: SessionRating)
  → ratingCount++, ratingSum += score
