## 一、产品概述

### 1.1 产品定位
智能体协作平台是一个支持多智能体协同工作的AI应用平台，用户可以通过配置智能体和技能（Skill），实现复杂任务的自动化处理。平台分为**管理后台**和**用户前台**两大模块。

### 1.2 核心价值
- **管理后台**：为管理员提供智能体和技能的配置管理能力
- **用户前台**：为终端用户提供智能、便捷的对话交互体验
- **协作机制**：支持单智能体和多智能体协同两种工作模式

### 1.3 用户角色
- **管理员**：使用后台管理系统，配置智能体、技能、知识库等
- **终端用户**：使用前台对话界面，与智能体交互完成任务

## 三、后台管理系统

### 3.1 Skill技能模块

#### 3.1.1 模块定位
Skill（技能）是智能体可调用的能力单元，用于执行特定功能（如网页搜索、文件读取、代码执行等）。

#### 3.1.2 Skill列表页

**页面路径：** `/skill`

**核心功能：**

1. **Skill展示**
   - 以卡片形式展示所有Skill
   - 每个卡片显示：图标、名称、描述、状态（草稿/已发布）、创建/更新时间
   - 支持按状态筛选（全部/草稿/已发布）

2. **搜索与筛选**
   - 顶部搜索框：支持按Skill名称模糊搜索
   - 状态筛选：可选择查看不同状态的Skill

3. **操作入口**
   - 【新建Skill】按钮：跳转到Skill编辑器（创建模式）
   - 卡片操作菜单：
     - 编辑：跳转到Skill编辑器（编辑模式）
     - 预览：在前台预览该Skill
     - 删除：删除该Skill（需二次确认）
     - 发布/下架：切换Skill状态

**交互流程：**
```
进入Skill列表
  → 浏览所有Skill
  → 点击【新建Skill】→ 跳转到Skill编辑器（创建模式）
  → 点击卡片【编辑】→ 跳转到Skill编辑器（编辑模式）
  → 点击【预览】→ 跳转到前台预览
```

### 3.2 智能体管理模块

#### 3.2.1 模块定位
智能体（Agent）是平台的核心执行单元，分为**单智能体模式**和**多智能体协同模式**两种类型。

#### 3.2.2 智能体列表页

**页面路径：** `/agent`

**核心功能：**

1. **智能体展示**
   - 列表/卡片两种视图模式
   - 每个智能体显示：图标、名称、类型标签（单智能体/多智能体）、描述、状态、创建时间
   - 多智能体类型带有"多智能体"标识Tag

2. **筛选与搜索**
   - 类型筛选：全部/单智能体/多智能体
   - 状态筛选：草稿/已发布
   - 分类筛选：按业务类别（行业分析/编程开发/数据分析/内容创作等）
   - 搜索框：按名称模糊搜索

3. **创建入口**
   - 【新建智能体】按钮，展开选项：
     - 自主规划Agent（单智能体）→ 跳转到单智能体配置页
     - 多智能体协同Agent → 跳转到多智能体配置页

4. **操作功能**
   - 编辑：跳转到对应配置页
   - 删除：删除智能体（需二次确认）
   - 发布/下架：切换智能体状态
   - 预览：在前台预览该智能体

**智能体数据结构（示例）：**
```
智能体 = {
  id: string,
  name: string,
  icon: emoji,
  type: 'autonomous' | 'collaborative',
  category: string,  // 分类
  description: string,
  status: 'draft' | 'published',
  isMultiAgent: boolean,  // 是否为多智能体
  multiAgentMembers: Agent[],  // 多智能体成员列表（仅多智能体类型有值）
  skills: Skill[],  // 配置的技能列表
  prompt: string,  // 提示词
  createTime: timestamp,
  updateTime: timestamp
}
```

#### 3.2.3 单智能体配置页（自主规划Agent）

**页面路径：** `/agent-editor?type=autonomous&id=xxx`

**页面布局：**
- 左侧：配置面板
- 右侧：实时预览

**核心功能：**

1. **基本信息**
   - 智能体名称（必填）
   - 智能体图标（emoji选择）
   - 智能体描述（必填）
   - 智能体分类（下拉选择）

2. **能力配置**
   - **模型选择**：DeepSeek-V3、GPT-4、Claude等
   - **提示词编辑器**：
     - 系统提示词（System Prompt）
     - 支持模板库
     - 支持变量插入
   - **Skill配置**：
     - 从Skill库中选择可用技能
     - 支持多选
     - 显示已选Skill列表
     - 可调整Skill优先级

3. **高级配置**
   - 上下文窗口大小
   - 温度系数（Temperature）
   - 最大输出长度
   - 是否启用深度思考

4. **知识库关联**
   - 选择关联的知识库
   - 配置检索策略

5. **保存与发布**
   - 【保存草稿】
   - 【保存并发布】
   - 【预览】：跳转到前台测试
   - 【返回】

**交互流程：**
```
进入单智能体配置页
  → 填写基本信息
  → 选择模型
  → 编辑提示词
  → 从Skill库选择技能（多选）
  → 配置高级参数
  → 关联知识库
  → 点击【预览】→ 在前台测试智能体
  → 点击【保存并发布】→ 保存配置，返回列表页
```

#### 3.2.4 多智能体协同配置页

**页面路径：** `/agent-collaborative`

**页面布局：**
- 左侧：应用配置面板
- 中间：Agent配置面板（主Agent + 子Agent列表）
- 右侧：预览与调试面板

**核心功能：**

1. **应用设置**（左侧面板）
   - **基本���息**：
     - 应用名称
     - 应用图标
     - 应用描述
   - **开场白**：
     - 用户进入时的欢迎消息
   - **推荐问**：
     - 预设推荐问题（最多5个）
     - 可添加/删除

2. **主Agent配置**（中间面板）
   - **角色定位**：规划Agent（负责任务拆解和分配）
   - **模型选择**：选择主Agent使用的AI模型
   - **参考上下文数**：控制历史对话轮数（1-10轮）
   - **提示词配置**：
     - 任务拆解能力
     - 子Agent调度逻辑
     - 结果整合规则

3. **子Agent配置**（中间面板）
   - **添加子Agent**：
     - 从现有Agent库选择（需要先创建为单智能体）
     - 显示官方推荐的子Agent
     - 支持搜索筛选
   - **子Agent列表**：
     - 显示已添加的子Agent（最多10个）
     - 每个子Agent显示：图标、名称、描述、官方标识
     - 启用/禁用开关
     - 删除按钮
   - **创建新子Agent**：
     - 点击【创建】→ 提示需要先创建单智能体
     - 引导跳转到单智能体配置页

4. **预览与调试**（右侧面板）
   - 应用预览卡片（显示图标和名称）
   - 聊天输入框（模拟用户输入）
   - 调试日志（显示执行过程）

5. **保存与发布**
   - 【保存配置】：保存当前所有配置
   - 【发布】：发布多智能体应用，使其在前台可用

**协同工作机制：**
```
用户提问
  → 主Agent（规划Agent）分析任务
  → 拆解为多个子任务
  → 分配给不同子Agent执行
  → 收集子Agent结果
  → 整合生成最终答案
  → 返回给用户
```

**交互流程：**
```
进入多智能体配置页
  → 填写应用基本信息
  → 配置主Agent（模型、提示词、上下文）
  → 点击【添加子Agent】→ 弹出选择弹窗
    → 从列表选择子Agent → 添加到团队
    → 或点击【创建】→ 跳转到单智能体配置页
  → 配置每个子Agent的启用状态
  → 在右侧预览面板测试
  → 点击【保存配置】→ 保存
  → 点击【发布】→ 发布应用
```

```typescript
interface Skill {

  status: 'draft' | 'published';

  // 单智能体特有

  // 多智能体特有
  isMultiAgent: boolean;
  multiAgentMembers?: Agent[];

  updateTime: timestamp;

interface MultiAgentApp {
  id: string;
  name: string;

  description: string;

  mainAgent: {
    name: string;
    model: string;
    contextCount: number;
    prompt: string;
  };

  subAgents: {
    id: string;
    enabled: boolean;
  }[];

  status: 'draft' | 'published';
  createTime: timestamp;
  updateTime: timestamp;
}

// 对话消息
interface Message {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  agentId?: string;
  agentName?: string;
  timestamp: string;

  // 状态
  status?: 'thinking' | 'calling' | 'completed';
  thinking?: string;
  toolCalls?: ToolCall[];

  // 多智能体协作
  isCollaborationFlow?: boolean;
  collaborationPhases?: CollaborationPhase[];
}

// 协作阶段
interface CollaborationPhase {
  id: string;
  phase: 'planning' | 'execution' | 'summary';
  title: string;
  content: string;
  expanded: boolean;
  toolCalls?: ToolCall[];
}

// 工具调用
interface ToolCall {
  id: string;
  name: string;
  status: 'running' | 'completed';
  result?: string;
}
```

## 七、用户体验设计

### 7.1 视觉规范

#### 颜色系统
- **主色调**：紫色渐变（#6366F1 → #8B5CF6）
- **成功色**：绿色（#10B981）
- **警告色**：橙色（#F59E0B）
- **提示色**：黄色（#FCD34D）
- **文本色**：
  - 主文本：#333
  - 次文本：#666
  - 提示文本：#999

#### 间距规范
- 卡片间距：16px
- 内容边距：12-20px
- 按钮高度：40px
- 圆角：8-12px

### 7.2 交互反馈

1. **加载状态**
   - 使用Spin组件显示加载动画
   - 显示进度提示（如"执行中... 2/4"）

2. **成功反馈**
   - 操作成功后显示Toast提示
   - 绿色勾选图标

3. **错误处理**
   - 红色边框高亮错误字段
   - 显示错误提示信息

4. **过渡动画**
   - 卡片展开/折叠：0.3s缓动
   - 按钮悬停：颜色渐变
   - 页面切换：淡入淡出

### 7.3 响应式设计

- 支持PC端（最小宽度1280px）
- 侧边栏可折叠（宽度260px → 0px）
- 智能体中心网格自适应（最小450px/卡片）
