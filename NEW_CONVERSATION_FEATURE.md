# 新建对话和对话管理功能更新说明

## 版本 1.2.0 - 2026-01-24

### 新增功能

#### 1. 新建对话界面增强

**位置**: 欢迎界面输入框

**新增功能**:

##### 1.1 @智能体和群组
- 在输入框中输入 `@` 符号唤起选择面板
- 支持两个选项卡：
  - **智能体**: 浏览所有可用的智能体，按类别分组显示
  - **群组**: 浏览已创建的群组
- 可以同时@多个智能体和群组
- 混合对话支持（既可以@智能体，也可以@群组）
- 已选择的智能体和群组以彩色标签形式显示在输入框上方
- 标签可点击关闭删除

##### 1.2 按钮调整到输入框下方
所有功能按钮现在位于输入框下方，包括：

**左侧功能按钮**:
- **上传文件**: 点击可选择本地文件上传
- **深度思考**: 切换深度思考模式（按钮高亮表示已启用）
- **知识库**: 选择使用的知识库

**右侧控制按钮**:
- **模型选择**: 下拉选择AI模型
- **发送**: 发送消息并创建新对话

##### 1.3 文件上传功能
- 点击"上传文件"按钮选择文件
- 支持多文件上传
- 上传的文件以黄色标签形式显示在输入框上方
- 显示文件名和文件图标（📎）
- 可点击关闭删除上传的文件

##### 1.4 深度思考模式
- 点击"深度思考"按钮切换状态
- 启用时按钮变为紫色渐变主题色
- 禁用时显示为默认灰色
- 状态保存在对话配置中

#### 2. 对话管理系统

##### 2.1 对话类型
系统支持三种对话类型：
- **agent**: 只包含智能体的对话
- **group**: 群组对话
- **mixed**: 混合对话（同时包含智能体和群组）

##### 2.2 对话数据结构
```typescript
interface Conversation {
  id: string;
  title: string;               // 对话标题（取自第一条消息）
  type: 'agent' | 'group' | 'mixed';
  agents?: Agent[];            // 参与的智能体列表
  group?: AgentGroup;          // 如果是群组对话
  messages: Message[];         // 消息列表
  createTime: string;
  deepThinking?: boolean;      // 是否启用深度思考
  knowledgeBase?: string[];    // 使用的知识库
  uploadedFiles?: UploadedFile[]; // 上传的文件
}
```

##### 2.3 新建对话流程
1. 用户在欢迎界面输入消息
2. （可选）@智能体或群组
3. （可选）上传文件
4. （可选）启用深度思考
5. （可选）选择知识库
6. 点击发送按钮
7. 系统创建新对话并进入对话界面
8. 被@的智能体开始回复

##### 2.4 对话界面统一
- 群组对话和普通对话使用统一的界面
- 对话头部显示：
  - 图标（群组图标或用户图标）
  - 标题（群组名称或对话标题）
  - 成员数量
  - 退出对话按钮
- 显示所有参与的智能体标签
- 消息列表显示所有对话历史
- 输入框支持@智能体和@all

#### 3. 新建对话和新建群组按钮优化

**功能增强**:
- 在对话状态下也可以点击"新建对话"按钮
- 点击后退出当前对话，返回欢迎界面
- 清空输入框和已选择的智能体
- "新建群组"按钮在任何状态下都可以使用
- 填写并创建群组后不会打断当前对话

#### 4. @智能体面板统一

**欢迎界面的@面板**:
- 显示所有智能体（按类别分组）
- 显示所有群组
- 支持智能体和群组混选

**对话界面的@面板**:
- 只显示当前对话中的参与者
- 支持@all功能
- 支持@单个智能体

### UI/UX改进

- **群组标签**: 蓝色背景，带团队图标
- **智能体标签**: 使用智能体的主题色，带emoji图标

#### 新增函数
1. `handleCreateConversation()`: 从欢迎界面创建新对话
2. `handleNewConversation()`: 点击新建对话按钮
3. `handleFileUpload(e)`: 处理文件上传
4. `handleSelectGroup(group)`: 选择群组
5. `handleSelectAgent(agent)`: 在新建对话时选择智能体
6. `simulateAgentReplyInConversation()`: 在对话中模拟智能体回复

#### 修改的函数
1. `handleSendMessage()`: 支持currentConversation
2. `handleInputChange()`: 支持在不同界面显示不同的@面板
3. `handleMentionAll()`: 支持currentConversation

### 使用场景

#### 场景1：创建单个智能体对话
1. 在欢迎界面输入消息
2. 输入 `@` 选择一个智能体
3. 点击发送
4. 进入对话界面，智能体开始回复

#### 场景2：创建群组对话
1. 在欢迎界面输入消息
2. 输入 `@` 切换到"群组"标签
3. 选择一个群组
4. 点击发送
5. 进入群组对话界面

#### 场景3：创建混合对话
1. 在欢迎界面输入消息
2. 输入 `@` 选择2个智能体
3. 再次输入 `@` 切换到"群组"标签，选择1个群组
4. 点击发送
5. 进入包含所有选中智能体的对话界面

#### 场景4：上传文件并深度思考
1. 在欢迎界面点击"上传文件"
2. 选择文件
3. 点击"深度思考"按钮启用
4. 输入消息并@智能体
5. 点击发送
6. 创建的对话会保存这些配置

#### 场景5：对话中新建对话
1. 正在某个群组对话中
2. 点击左侧边栏的"新建对话"按钮
3. 退出当前对话，返回欢迎界面
4. 创建新的对话
