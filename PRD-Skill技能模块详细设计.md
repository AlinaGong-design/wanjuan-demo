## 一、模块概述

### 1.1 产品定位

Skill（技能）是智能体协作平台的**核心能力单元**，是连接用户需求与AI执行能力的桥梁。每个Skill封装了一项特定的功能能力（如网页搜索、文件读取、代码执行、数据分析等），可被智能体调用以完成复杂任务。

**核心理念**：
- **低代码配置**：管理员通过可视化界面配置Skill，无需编写代码
- **即插即用**：Skill可被多个智能体复用，降低配置成本
- **能力扩展**：通过Skill机制，智能体可以不断扩展新能力

### 1.2 在平台架构中的角色

```
用户需求
    ↓
智能体（Agent）
    ↓
调用 → Skill技能库 ← 配置管理
    ↓
工具/API执行
    ↓
返回结果
```

**价值链条**：
1. **对管理员**：通过Skill编辑器，将各种工具能力标准化封装
2. **对智能体**：通过选择Skill，快速获得执行能力
3. **对终端用户**：享受智能体提供的多样化服务

### 1.3 业务目标

| 目标维度 | 具体指标 |
|---------|---------|
| **能力丰富度** | 平台提供50+官方Skill，覆盖常见业务场景 |
| **配置效率** | 管理员创建一个Skill平均耗时 < 10分钟 |
| **复用率** | 单个Skill平均被3+个智能体使用 |
| **可用性** | Skill测试通过率 > 95% |

### 3.3 功能模块详解

#### 3.3.1 Skill卡片展示

**设计规范**：

| 元素 | 说明 | 规格 |
|------|------|------|
| **图标** | emoji表情 | 48x48px，居中显示 |
| **名称** | Skill名称 | 最多20字符，超出显示省略号 |
| **描述** | Skill功能描述 | 最多60字符，2行显示，超出截断 |
| **状态标签** | 草稿/已发布 | Tag组件，草稿=橙色，已发布=绿色 |
| **时间信息** | 创建/更新时间 | 灰色小字，相对时间（如"3小时前"） |

**卡片状态**：
- **默认状态**：白色背景，浅灰边框
- **悬停状态**：边框变为品牌色（#6366F1），轻微阴影
- **已选中状态**（多选场景）：背景色变浅蓝色

**卡片操作菜单**：
- 点击卡片右上角「⋮」图标展开
- 操作项：
  1. ✏️ **编辑**：跳转到编辑模式
  2. 👁️ **预览**：在前台预览效果
  3. 🗑️ **删除**：二次确认后删除
  4. 📤 **发布** / 📥 **下架**：切换状态

#### 3.3.2 搜索与筛选

**搜索框**：
- **位置**：页面顶部左侧
- **交互**：
  - 占位符：「搜索Skill名称...」
  - 实时搜索：输入后300ms防抖触发
  - 支持中英文模糊匹配
  - 清空按钮：点击×清除搜索词
- **搜索范围**：仅匹配Skill名称字段

**状态筛选**：
- **选项**：
  1. 全部（默认）
  2. 草稿
  3. 已发布
- **交互**：
  - 下拉选择器形式
  - 选择后立即刷新列表
  - 状态数量实时显示（如「草稿(5)」）

**组合筛选**：
- 搜索 + 状态筛选可同时生效
- 筛选结果实时显示数量：「共找到 8 个Skill」

#### 3.3.3 新建Skill入口

**按钮规格**：
- **位置**：页面右上角
- **样式**：主按钮（Primary），渐变色背景
- **文案**：「+ 新建Skill」
- **图标**：加号图标

**交互流程**：
```
点击【新建Skill】
  → 跳转到 /skill-editor?mode=create
  → 进入创建模式的Skill编辑器
  → 所有字段为空，等待填写
```

#### 3.3.4 批量操作（高级功能）

**选择模式**：
- 点击左上角「☑️ 选择」进入批量模式
- 卡片左上角出现复选框
- 底部出现操作栏

**批量操作栏**：
```
┌─────────────────────────────────────────────────┐
│ 已选择 5 个Skill  [取消选择]  [批量发布] [批量删除] │
└─────────────────────────────────────────────────┘
```

**操作规则**：
- 批量发布：仅对草稿状态的Skill生效
- 批量删除：需二次确认，显示「即将删除 X 个Skill，确认？」

### 4.3 基础信息配置

#### 4.3.1 Skill名称

**字段规格**：

| 属性 | 规则 |
|------|------|
| 字段类型 | 单行文本输入框 |
| 必填项 | ✅ 是 |
| 最大长度 | 50字符 |
| 字符计数 | 实时显示（如「25/50」） |
| 验证规则 | 不能为空，不能只包含空格 |
| 错误提示 | 「Skill名称不能为空」 |

**命名建议**（浮层提示）：
- ✅ 推荐：「网页搜索」「文档分析」「Python代码执行」
- ❌ 避免：「Skill1」「测试」「ABC」
- 提示文案：「使用清晰的动词+名词结构，让用户一眼看懂功能」

#### 4.3.2 Skill图标

**选择方式**：

**方式1：Emoji选择器（推荐）**
- 点击输入框弹出Emoji面板
- 分类展示：常用、笑脸、动物、物品、符号等
- 搜索功能：输入「搜索」可快速定位🔍
- 最近使用：显示最近选择的10个emoji

**方式2：直接输入**
- 支持粘贴emoji字符
- 自动识别并渲染

**默认图标**：
- 若未选择，默认显示「📦」

**预览**：
- 实时预览区同步显示选择的图标
- 大尺寸预览（64x64px）

#### 4.3.3 Skill描述

**字段规格**：

| 属性 | 规则 |
|------|------|
| 字段类型 | 多行文本输入框（TextArea） |
| 必填项 | ✅ 是 |
| 最大长度 | 200字符 |
| 行数 | 自适应高度，最少3行，最多8行 |
| 验证规则 | 不能为空，至少10个字符 |
| 错误提示 | 「请输入至少10个字的Skill描述」 |

**撰写指导**（占位符文案）：
```
请描述这个Skill的功能和使用场景，例如：
「该Skill可以搜索互联网上的实时信息，支持关键词搜索、新闻搜索等多种模式，适用于需要获取最新资讯的场景」
```

**AI辅助生成按钮**（高级功能）：
- 位置：描述输入框右上角
- 功能：根据Skill名称自动生成描述建议
- 交互：
  1. 点击「✨ AI生成描述」按钮
  2. 调用AI接口，传入Skill名称
  3. 1-2秒后返回3条描述建议
  4. 用户选择一条插入，或手动编辑

#### 4.3.4 Skill分类

**分类体系**：

| 分类 | 说明 | 示例Skill |

**选择方式**：
- 下拉选择器
- 单选模式
- 默认值：「信息获取」

**未来扩展**：

### 4.4 能力配置（核心模块）

- 支持高级Markdown语法和模板变量

**编辑器规格**：

| 属性 | 规则 |
|------|------|
| 编辑器类型 | 代码编辑器（Monaco Editor或类似） |
| 格式支持 | Markdown语法高亮 |
| 最大长度 | 10,000字符 |
| 自动保存 | 每30秒自动保存草稿 |

**模板库功能**：

点击「📚 模板库」按钮，弹出模板选择面板：

**分类模板**：

```markdown
你是一个专业的网页搜索助手。

1. 接收用户输入的搜索词

## 输出格式

1. **[标题1](链接1)**
   摘要内容...

2. **[标题2](链接2)**
   摘要内容...
```

2. **文档处理类模板**
```markdown
你是一个文档处理专家。

## 任务目标
读取用户上传的文档，进行内容分析和提取。

## 支持格式
- PDF
- Word (.doc, .docx)
- Markdown (.md)
- 纯文本 (.txt)

1. 识别文档格式

5. 返回处理结果
```

- code: Python代码字符串

1. 验证代码安全性
2. 在沙箱环境中执行
3. 捕获标准输出和错误输出
4. 返回执行结果

## 输出格式

执行结果：
{输出内容}

执行时间：X秒
\```

#### 4.4.2 工具配置

**工具类型**：

1. **API调用工具**
   - 配置项：
     - API地址（URL）
     - 请求方法（GET/POST/PUT/DELETE）
     - 请求头（Headers）
     - 请求体（Body）
     - 超时时间（默认30秒）
   - 认证方式：
     - 无认证
     - API Key
     - OAuth 2.0
     - 自定义Token

2. **文件操作工具**
   - 支持操作：
     - 读取文件
     - 写入文件
     - 删除文件
     - 列出目录
   - 安全限制：
     - 仅允许操作指定目录（如/workspace）
     - 文件大小限制：单文件 < 50MB
     - 格式限制：可配置允许的文件扩展名

3. **代码执行工具**
   - 支持语言：
     - Python 3.x
     - JavaScript (Node.js)
     - Shell (Bash)
     - SQL（连接数据库）
   - 沙箱配置：
     - 执行超时：可配置（5-300秒）
     - 内存限制：可配置（128MB-2GB）
     - 网络访问：开启/关闭
     - 文件系统访问：开启/关闭

4. **数据库查询工具**
   - 支持数据库：
     - MySQL
     - PostgreSQL
     - MongoDB
     - Redis
   - 配置项：
     - 连接字符串
     - 账号密码
     - 默认数据库
     - 连接池大小

## 七、数据模型设计

### 7.1 Skill数据结构（示例）

```typescript
interface Skill {
  // 基础信息
  id: string;                    // 唯一标识
  name: string;                  // Skill名称
  icon: string;                  // emoji图标
  description: string;           // Skill描述
  category: string;              // 分类

  // 状态信息
  status: 'draft' | 'published'; // 状态
  createTime: timestamp;         // 创建时间
  updateTime: timestamp;         // 更新时间
  creatorId: string;             // 创建者ID

  // 能力配置
  prompt: string;                // 提示词内容
  tools: Tool[];                 // 工具配置列表
  inputParams: Parameter[];      // 输入参数定义
  outputFormat: OutputSchema;    // 输出格式定义

  // 使用统计（可选）
  usageCount: number;            // 被调用次数
  agentCount: number;            // 被多少个智能体使用
  successRate: number;           // 执行成功率
  avgExecutionTime: number;      // 平均执行时长（毫秒）
}

interface Tool {
  id: string;
  type: 'api' | 'file' | 'code' | 'database';
  name: string;
  config: ToolConfig;            // 具体配置（根据type不同而不同）
  enabled: boolean;
}

interface Parameter {
  name: string;
  type: 'string' | 'number' | 'boolean' | 'array' | 'object';
  required: boolean;
  defaultValue?: any;
  description: string;
  validation?: string;           // 验证规则（正则表达式等）
}

interface OutputSchema {
  format: 'json' | 'markdown' | 'text';
  schema?: object;               // JSON Schema定义
  example?: string;              // 输出示例
}
```

## 十二、附录

### 12.1 常见问题FAQ

**Q1：Skill和智能体有什么区别？**

A：
- **Skill（技能）**：是具体的功能能力，如"搜索网页"、"执行代码"，是工具。
- **智能体（Agent）**：是具备多种Skill的AI实体，可理解任务、规划执行、调用Skill完成任务。
- 关系：智能体通过配置多个Skill获得能力，一个Skill可被多个智能体复用。

**Q2：如何判断一个Skill配置得好不好？**

A：好的Skill应该满足：
1. **提示词清晰**：明确定义Skill的目标、步骤、输出格式
2. **参数定义完整**：输入参数有清晰的类型和验证规则
3. **工具配置正确**：API地址、认证方式、超时设置合理
4. **测试通过**：至少执行过一次成功测试
5. **可复用性高**：描述清晰，其他智能体也能理解使用

**Q3：Skill的提示词应该怎么写？**

A：推荐结构：
```
你是一个 [角色定位]

## 任务目标
[描述Skill要完成的任务]

## 输入参数
- 参数1：说明
- 参数2：说明

## 执行步骤
1. 步骤1
2. 步骤2
3. 步骤3

## 输出格式
[定义返回结果的格式]

## 注意事项
[特殊处理逻辑、边界情况]
```

**Q4：为什么我的Skill测试失败了？**

A：常见原因：
1. **API配置错误**：检查API地址、认证信息是否正确
2. **参数格式不对**：检查输入参数类型是否匹配
3. **网络超时**：增加超时时间或检查网络连接
4. **权限不足**：检查API Key是否有足够权限
5. **提示词逻辑错误**：检查提示词是否有矛盾或不清晰的地方

查看「工具调用日志」可定位具体问题。

**Q5：一个Skill可以调用多个工具吗？**

A：可以。一个Skill可以配置多个工具，根据任务需要依次或并发调用。例如：
- 先调用「web_search」工具搜索信息
- 再调用「text_analysis」工具分析搜索结果
- 最后调用「markdown_generator」工具格式化输出
