## 一、背景与目标

### 1.1 背景

Skill 是平台的核心能力单元，承载了从代码编写、调试到发布的完整生命周期。当前版本在交互流畅度、文件能力、协作共享、权限管控等方面存在明显短板，用户操作路径冗长，协作场景缺失，制约了平台向团队级工具演进。

### 1.2 本期目标

| 目标 | 度量指标 |
|---|---|
| 缩短用户进入编辑的路径 | 列表→编辑从 2 步缩短为 1 步 |
| 完善文件处理能力 | 生成侧与调试侧均支持上传文件 |
| 建立版本与权限管理机制 | 发布时可选版本号与可见范围 |
| 引入共享协作空间 | 团队/公开 Skill 可在共享空间被发现和引用 |
| 提升 Skill 组织效率 | 支持标签体系，可自定义管理 |
| 优化导入体验 | 导入过程状态可视化，用户无需等待黑盒 |

## 四、功能详细说明

### 4.1 Skill 列表点击直达编辑

#### 4.1.1 需求背景

现有交互：用户在列表卡片上找到编辑图标按钮（EditOutlined）点击后进入编辑页。对于高频编辑用户，每次操作都需要精确点击小图标，认知负担高，效率低。

#### 4.1.2 交互变更说明

| 区域 | 旧交互 | 新交互 |
|---|---|---|
| 卡片主体区域（图标+名称+描述） | 无响应 | 单击 → 直接进入编辑页 |
| 卡片操作按钮区 | 编辑/发布/历史/删除 图标并列 | 保留，不变 |

**细节要求：**

- 卡片主体区域 hover 状态需有视觉反馈（边框高亮或背景浅色过渡），让用户感知该区域可点击
- 操作按钮区（右上角或底部操作栏）点击事件需做 `stopPropagation`，避免触发卡片跳转
- 键盘可访问性：卡片主体区域支持 Tab 聚焦 + Enter 触发跳转
- 跳转目标：`#skill-editor?mode=edit&id={skillId}`

#### 4.1.3 边界情况

| 场景 | 处理方式 |
|---|---|
| 共享空间中他人的 Skill | 点击进入只读预览模式，顶部 Banner 提示"仅预览，无法编辑" |
| Skill 状态为 offline | 正常进入编辑页，编辑器内提示当前已下线 |

### 4.2 Skill 生成侧上传文件

#### 4.2.1 需求背景

在 Skill 编辑器的代码生成阶段，开发者常需要上传参考文档（如 API 说明书、示例数据文件）辅助 AI 生成更准确的代码。当前版本无此入口，用户需手动拷贝内容，效率低。

#### 4.2.2 功能说明

**入口位置**：SkillEditor 代码生成区（左侧对话/生成面板）的输入框附近，增加文件上传按钮（PaperClipOutlined 图标）。

**支持能力：**

| 项目 | 说明 |
|---|---|
| 支持格式 | `.txt` `.md` `.pdf` `.json` `.yaml` `.csv` `.py` `.js` `.ts` 等文本/代码类文件 |
| 单文件大小上限 | 10 MB |
| 单次上传数量 | 最多 5 个文件 |
| 展示方式 | 输入框上方以文件标签（Tag）形式展示，可逐个删除 |

**处理逻辑：**

- 文件内容作为���下文注入 Skill 生成 prompt，AI 可引用文件内容生成代码
- 文件仅在当前生成会话中有效，不会持久化为 Skill 项目文件
- 若需持久化，用户可在生成后通过文件树的"上传文件"功能手动存入

**状态与反馈：**

| 状态 | 视觉表现 |
|---|---|
| 上传中 | Tag 内 Loading 图标 + 文件名 |
| 上传成功 | Tag 内文件图标 + 文件名 + 删除按钮 |
| 上传失败 | Tag 变红 + 错误 Tooltip + 重试按钮 |
| 超出限制 | Toast 提示具体原因（格式/大小/数量） |

---

### 4.3 预览调试侧上传文件

#### 4.3.1 需求背景

调试阶段开发者需要以真实数据测试 Skill，当 Skill 涉及文件解析、图片处理、数据分析等场景时，无法上传文件验证，调试完整性严重不足。

#### 4.3.2 功能说明

**入口位置**：SkillEditor 右侧"调试与测试"面板的消息输入框，增加文件上传按钮（与生成侧一致的 PaperClipOutlined 图标）。

**与生成侧的差异：**

| 对比项 | 生成侧 | 调试侧 |
|---|---|---|
| 用途 | 辅助 AI 生成代码 | 作为真实 Skill 运行时输入 |
| 文件是否参与执行 | 否（仅作上下文） | 是（传递给 Skill 运行时） |
| 会话持久性 | 本次生成会话 | 单轮调试消息 |

**支持格式**：与 Skill 本身声明的 `input_types` 保持一致；若 Skill 未声明，则允许常见格式。

**调试结果展示：**

- 若 Skill 返回文件类型（如生成的图片、PDF），在结果区域内联展示（图片直接渲染，其他文件提供下载链接）

### 4.4 Skill 发布版本与权限范围

#### 4.4.1 需求背景

现有 Skill 缺乏规范化的版本管理，团队无法区分生产版本与开发版本，也无法控制 Skill 的可见性。引入语义化版本号和可见范围，是 Skill 走向团队协作的基础。

#### 4.4.2 发布流程

触发方式：编辑器顶部操作栏点击"发布"按钮，弹出发布弹窗。

**发布弹窗字段：**

| 字段 | 类型 | 说明 |
|---|---|---|
| 版本号 | 输入框（只读+可手动覆盖） | 系统根据上一版本自动建议下一个版本号，支持 Patch/Minor/Major 三种递增方式快捷选择 |
| 更新说明 | 多行文本 | 必填，描述本次发布的变更内容，最多 500 字 |
| 可见范围 | 单选 | 见下方说明 |

**可见范围（VisibilityScope）：**

| 选项 | 图标 | 说明 |
|---|---|---|
| 仅自己（private） | LockOutlined | 仅发布者本人可见和使用 |
| 团队内（team） | TeamOutlined | 同组织内所有成员可见，出现在共享空间 |
| 公开（public） | GlobalOutlined | 平台全局可见，出现在共享空间 |

**版本号建议规则：**

```
当前版本 1.2.3
  Patch → 1.2.4（Bug 修复/小调整）
  Minor → 1.3.0（新增功能，向下兼容）
  Major → 2.0.0（破坏性变更）
```

**发布后：**

- Skill 状态变为 `published`
- 版本历史记录新增一条发布事件
- 若可见范围为 team/public，共享空间实时更新

#### 4.4.3 版本历史查看

入口：列表卡片操作区的 HistoryOutlined 图标，或编辑器内"历史"按钮。

**历史 Drawer 展示内容：**

- 事件类型（保存 / 发布）
- 时间、操作人
- 版本号（仅发布事件有）
- 更新说明
- 可见范围
- 操作：「查看该版本」「回滚至该版本」

**回滚逻辑：**

- 回滚会将该版本快照覆盖当前草稿
- 操作前弹出确认弹窗，说明"此操作不可撤销，当前草稿将被覆盖"
- 回滚后 Skill 状态变为 `draft`，需重新发布才能生效

### 4.5 共享空间

#### 4.5.1 需求背景

当前所有 Skill 仅在"我的 Skill"Tab 中可见，团队成员无法发现和复用其他人发布的 Skill，协作价值无法释放。

#### 4.5.2 功能说明

**入口位置**：Skill 列表页顶部 Tabs，增加"共享空间"Tab（与"我的 Skill"并列）。

**内容范围：**

| 来源 | 显示条件 |
|---|---|
| 他人发布的 Skill | 可见范围为 `team`（同组织）或 `public` |
| 自己发布的 Skill | 可见范围为 `team` 或 `public` |

**共享空间卡片展示：**

与"我的 Skill"卡片一致，额外展示：
- 发布者头像 + 名称
- 最新发布版本号
- 使用人数（若有统计）

**可执行操作：**

| 操作 | 说明 |
|---|---|
| 查看详情 | 点击卡片 → 只读编辑器视图 |
| 引用到 Agent | 一键将该 Skill 关联到用户自己的 Agent |
| Fork（复制到我的 Skill） | 将共享 Skill 复制一份到"我的 Skill"，可二次编辑 |

**共享空间筛选：**

- 搜索（名称关键字）
- 分类筛选（与我的 Skill 筛选项一致）
- 发布者筛选（全部 / 仅团队成员 / 仅公开）
- 标签筛选

### 4.6 标签与管理标签

#### 4.6.1 需求背景

随着 Skill 数量增长，分类维度单一（仅靠 category 字段），用户难以快速定位所需 Skill。标签体系提供更灵活的多维度组织能力。

#### 4.6.2 标签操作

**入口：** Skill 列表卡片底部"添加标签"按钮（TagOutlined 图标）。

**标签下拉面板：**

| 功能 | 说明 |
|---|---|
| 搜索已有标签 | 实时过滤，关键字高亮 |
| 创建新标签 | 搜索无结果时，展示"创建 '{keyword}' 标签"选项，点击即创建并自动选中 |
| 切换选中状态 | 点击标签 Toggle 选中/取消，已选标签显示 CheckOutlined 图标 |
| 管理标签入口 | 面板底部"管理标签"按钮 → 打开全局管理弹窗 |

**多 Skill 共用一套全局标签库**，标签与 Skill 为多对多关系。

#### 4.6.3 管理标签弹窗（ManageTagsModal）

| 操作 | 说明 |
|---|---|
| 查看全部标签 | 列表展示，每条显示标签名 + 关联 Skill 数 |
| 新增标签 | 输入标签名 → 确认创建 |
| 编辑标签名 | 行内编辑，修改后保存 |
| 删除标签 | 确认弹窗，删除后自动解除与所有 Skill 的关联 |

**标签展示位置：**

- Skill 列表卡片底部（最多显示 3 个，超出显示 +N）
- Skill 编辑器 Basic Info 区域
- 共享空间卡片
- 列表页顶部筛选栏（可按标签筛选）

**标签命名规则：**

- 长度：2–20 个字符
- 不允许重复（大小写不敏感）
- 不允许纯空格

### 4.7 导入 Skill 状态可视化

#### 4.7.1 需求背景

用户上传 Skill 包（.zip / .skill / .tar.gz）后，当前系统直接进入黑盒等待，用户不知道导入进行到哪一步，容易焦虑或误以为卡死，体验差。

#### 4.7.2 状态展示方案

**展示方式**：导入触发后，整个 Skill 列表页主内容区被「导入进度覆盖层」替代（全页状态，非弹窗），完成后自动消失并刷新列表。

**进度覆盖层结构：**

```
┌─────────────────────────────────────┐
│  正在导入 Skill 包：xxx.skill        │
│                                     │
│  ● 解压文件包          ✓ 完成        │
│  ● 读取 SKILL.md       ✓ 完成        │
│  ● 解析依赖配置        ⟳ 进行中...   │
│  ● 初始化运行环境      ○ 等待中      │
│  ● 注册 Skill 元信息   ○ 等待中      │
│                                     │
│  [取消导入]                          │
└─────────────────────────────────────┘
```

**各阶段状态定义：**

| 阶段 | 图标 | 说明 |
|---|---|---|
| 等待中 | ○（灰色） | 尚未开始 |
| 进行�� | SyncOutlined（蓝色旋转） | 正在执行 |
| 完成 | CheckCircleOutlined（绿色） | 成功 |
| 失败 | CloseCircleOutlined（红色） | 出错 |

**导入阶段列表（顺序执行）：**

1. 解压文件包
2. 读取 SKILL.md（校验格式、提取元信息）
3. 解析依赖配置（requirements.txt 或 package.json）
4. 初始化运行环境
5. 注册 Skill 元信息（写入系统）

**导入成功：**

- 覆盖层显示全绿 + "导入成功" + 3 秒后自动消失
- 新 Skill 出现在"我的 Skill"列表顶部，状态为 `draft`

**导入失败：**

- 失败阶段高亮红色，展示错误信息（如"SKILL.md 格式不合法，缺少 name 字段"）
- 提供「查看日志」（展开详细错误文本）和「重新上传」按钮

**取消导入：**

- 点击"取消导入"弹出确认弹窗
- 确认后终止进程，清理临时文件，恢复列表页

## 六、交互规范

### 6.1 卡片点击跳转

- 卡片可点击区域：图标 + 名称 + 描述三行，不含底部标签区和操作按钮区
- Hover 样式：`box-shadow` 加深 + `cursor: pointer`
- 过渡动画：`transition: box-shadow 0.2s ease`

### 6.2 文件上传通用规范

- 上传按钮统一使用 PaperClipOutlined 图标
- 支持点击选择和拖拽两种方式
- 超出大小/格式限制立即提示，不进入队列
- 上传中禁止重复点击同一按钮

### 6.3 版本号输入

- 格式校验：`^\d+\.\d+\.\d+$`
- 不允许小于当前最高版本号
- 自动建议不强制采用，允许用户手动输入

### 6.4 标签

- 标签 Tag 颜色：使用系统预设调色板（蓝/绿/橙/紫/红等），创建时自动分配，可选择不修改
- 标签最多显示数：卡片上限 3 个，超出显示 `+N` Tooltip 展示全部

## 七、异常与边界处理

| 场景 | 处理方式 |
|---|---|
| 网络中断时点击发布 | 弹窗禁用确认按钮 + 提示"网络异常，请检查连接" |
| 导入文件损坏 | 第一阶段（解压）即失败，给出明确错误信息 |
| 标签名与已有标签重复 | 输入框下方红色提示"标签已存在" |
| 共享空间加载失败 | Tab 内显示错误状态 + 重试按钮 |
| Fork 时目标 Skill 已被删除 | Toast 提示"该 Skill 已不存在，无法 Fork" |
| 回滚版本不存在 | Toast 提示"目标版本已失效" |
| 版本号低于已发布最高版本 | 发布弹窗内校验，红色错误提示，禁止提交 |

## 八、性能要求

| 指标 | 要求 |
|---|---|
| 文件上传响应时间 | 10MB 文件 ≤ 30s，需有进度条 |
| 列表卡片跳转 | 点击后 ≤ 300ms 开始路由跳转 |
| 共享空间首屏加载 | ≤ 2s（含分页，默认 20 条） |
| 导入进度状态刷新 | ≤ 1s 轮询间隔 |
| 标签搜索响应 | 输入后 ≤ 200ms 过滤结果 |

## 十、验收标准

### 10.1 Skill 列表点击直达编辑
- [ ] 点击卡片主体（图标/名称/描述区域）跳转到编辑页
- [ ] 操作按钮区点击不触发跳转
- [ ] Hover 状态有视觉反馈
- [ ] 共享空间中他人 Skill 进入只读模式

### 10.2 生成侧上传文件
- [ ] 输入框附近有文件上传入口
- [ ] 支持规定格式，超出格式/大小有错误提示
- [ ] 文件以 Tag 形式展示，可删除
- [ ] 文件内容注入生成上下文

### 10.3 调试侧上传文件
- [ ] 调试输入框附近有文件上传入口
- [ ] 文件作为运行时输入传递给 Skill
- [ ] 文件类型输出可内联展示或下载

### 10.4 发布版本与权限范围
