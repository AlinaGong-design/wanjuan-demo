## 一、需求背景

### 1.1 业务现状

AI Agent 管理平台作为企业 AI 资产的统一管理入口，承载智能体、技能、API 插件、评测规则等核心配置资产的创建、迭代与运营全流程。随着平台使用规模扩大、多人协作场景增多，以下问题日益突出：

**问题一：配置变更不可追溯**

智能体提示词、技能代码、API Schema 等配置一经修改即覆盖原有内容，无历史记录。线上问题发生后无法确认「改了什么、谁改的、改之前是什么状态」，排查效率极低。

**问题二：发布动作无版本隔离**

当前「保存即生效」机制导致草稿与正式配置混用，开发调试中的临时改动可能直接影响线上运行的智能体，存在稳定性隐患。

**问题三：历史状态无法恢复**

误操作（如错误清空提示词、误修改 Schema）发生后没有回退手段，只能人工重新填写，成本高且易出错。

**问题四：资产访问范围不可控**

所有资产对平台用户全量可见，个人测试中的草稿配置、团队内部评测集等均无隔离机制，存在数据泄漏与协作干扰风险。

### 1.2 用户诉求

| 用户角色 | 诉求 |
|---------|------|
| 智能体开发者 | 能看到每次发布了什么，出问题时快速还原到上一个稳定版本 |
| 团队负责人 | 控制哪些资产对外可见，团队内资产不被外部随意访问 |
| 评测人员 | 评测任务区分「我的」和「团队共享」，操作权责清晰 |
| 平台管理员 | 所有资产变更留有记录，满足内部合规要求 |

## 二、核心价值

本期功能围绕「**变更可追溯、发布可管控、历史可恢复、访问可隔离**」四个核心价值展开：

### 价值一：变更可追溯

为五类配置资产（智能体、技能、API 插件、评测集、评估器）引入**历史事件时间线**，每一次草稿保存与正式发布均留存记录，包含操作人、时间、变更摘要。

### 价值二：发布可管控

建立「**草稿 → 正式版本**」两阶段工作流。草稿修改不影响已发布版本；发布时显式填写版本号与变更日志，生成不可变快照，版本之间严格隔离。

### 价值三：历史可恢复

基于已发布版本的快照，支持**版本预览**（只读浏览历史配置）与**一键回滚**（将草稿内容覆写为指定历史版本），误操作可在 30 秒内恢复。

### 价值四：访问可隔离

引入三级**可见范围（Scope）**控制机制，精细管理资产的访问权限。评测任务额外增加**创建人操作权限**限制，实现操作责任归属清晰化。

## 四、详细需求设计

### 4.1 基础概念定义

| 概念 | 定义 |
|------|------|
| **草稿（Draft）** | 当前正在编辑、尚未正式发布的状态。对外不可见，可随时修改。 |
| **版本快照（Snapshot）** | 发布时对当前配置的全量存档，生成后不可更改。是回滚和预览的数据来源。 |
| **历史事件（History Event）** | 每一次「草稿保存」或「正式发布」操作在时间线上生成的一条记录。 |
| **版本号** | 语义化格式 `vX.Y.Z`，由用户填写，系统根据上次版本自动建议。 |
| **版本预览** | 临时切换到历史快照的只读浏览状态，不影响当前草稿内容。 |
| **回滚** | 将历史快照内容覆写到当前草稿，使草稿恢复到目标历史状态。 |

**资产保存状态流转：**

```
新建  ──[保存]──▶  草稿(draft)  ──[发布]──▶  已发布(published)
                      ▲                              │
                      └──────────[修改任意字段]───────┘
```

**可见范围（VisibilityScope）三级定义：**

| 值 | 中文 | 适用场景 |
|----|------|---------|
| `private` | 仅自己 | 个人调试、测试版本，不对外暴露 |
| `team` | 团队内 | 团队协作开发，仅团队成员可见 |
| `public` | 公开 | 正式对外，平台所有用户可查看 |

---

### 4.2 用户角色与权限

**角色定义：**

| 角色 | 描述 |
|------|------|
| 创建人（Owner） | 资产的创建者，拥有完整操作权限 |
| 团队成员（Member） | 同团队用户，可访问 `team` 和 `public` 范围的资产 |
| 公开访问者（Visitor） | 仅可访问 `public` 范围资产，无任何写操作权限 |

**操作权限矩阵：**

| 操作 | 创建人 | 团队成员 | 公开访问者 |

| 编辑配置 | ✅ | ❌ | ❌ |
| 草稿保存 | ✅ | ❌ | ❌ |
| 发布正式版本 | ✅ | ❌ | ❌ |
| 查看历史版本列表 | ✅ | ✅ (team/public) | ✅ (public) |
| 预览��史版本快照 | ✅ | ✅ (team/public) | ✅ (public) |

### 4.3 通用交互规范（F-01 ～ F-05）

以下四个交互组件被所有模块共享复用，各模块仅在字段内容上存在差异。

#### 4.3.1 历史版本抽屉（F-01）

**触发入口：** 编辑器页或列表页的「历史版本」按钮，仅当资产存在至少 1 条历史事件时展示该按钮。

**抽屉规格：** 宽度 440px（列表页模块）/ 480px（Agent 编辑器），从页面右侧滑出。

**内容结构：**

```
标题栏：  历史版本  [N 条记录]
─────────────────────────────────────────
草稿区：  ┌────────────────────────────┐
          │ 📄 草稿（当前编辑版本）      │ [当前]    ← 非预览态：绿色背景/边框
          └────────────────────────────┘
          （预览态：蓝色背景/边框，整块可点击返回草稿）

分隔线：  ──── N 条历史记录 ────

时间线：  ● 发布节点（完整版本卡片）
              v1.1.0  [正式发布]  [预览中?]
              提交人  ：xxx
              发布时间：2026-01-25 01:45
              权限范围：[团队内]
              ┌─版本说明──────────────────┐
              │ 新增数据分析技能，优化搜索策略 │  ← 紫色背景块
              └───────────────────────────┘
              [查看版本]  [回滚]

          ○ 保存节点（简洁单行）
              优化提示词与工具配置  [草稿保存]    xxx · 2026-01-22 14:00
```

**视觉规范：**

| 元素 | 样式 |
|------|------|
| 发布节点卡片背景 | `#ffffff`，`border: 1px solid #e8e8e8`，`box-shadow: 0 1px 4px rgba(0,0,0,0.04)` |
| 版本号文字 | `font-family: monospace`，`color: #6366F1`，`font-weight: 700` |
| 版本说明块 | `background: #f5f3ff`，`color: #6366F1`，`border-radius: 6px`，`padding: 8px 12px` |

#### 4.3.2 版本预览模式（F-02）

**进入方式：** 点击历史版本卡片「查看版本」按钮。

**页面顶部警告 Banner：**

```
┌──────────────────────────────────────────────────────────────────────┐
│  👁  当前浏览的是历史版本  v1.0.0  发布于 2026-01-20，由 xxx 提交      │
│                                                          [退出预览 ×] │
└──────────────────────────────────────────────────────────────────────┘
```

- Banner 背景色：`#fef3c7`，下边框：`1px solid #fcd34d`
- 版本号展示为橙色 Tag，字体 monospace

**预览态全局约束：**
- 所有表单输入框、下拉框、开关：`disabled={true}`，不可交互
- 发布按钮、保存按钮、编辑相关快捷操作：隐藏（不渲染）
- 「历史版本」按钮：保持可用，支持在预览态下切换查看其他历史版本
- 表单字段内容被替换为目标版本快照数据（不改变草稿底层状态）

**退出预览的三种方式：**
1. 点击 Banner 中「退出预览 ×」按钮
2. 点击历史版本抽屉中草稿区域
3. 点击预览 Banner 或卡片内的独立「退出」按钮（部分模块）

退出后表单内容恢复为当前草稿状态，`previewVersion` 状态置为 `null`。

#### 4.3.3 发布版本弹窗（F-03）

**触发：** 点击「发布」按钮。

**弹窗字段：**

| 字段 | 控件 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| 版本号 | Input | ✅ | 系统自动建议 | 格式严格为 `vX.Y.Z` |
| 版本说明 / 变更日志 | Textarea | ❌ | 空 | 最大 500 字 |
| 公开范围 | Select 或 Radio | ✅ | `team` | private / team / public |

**部分模块（评测集、评估器）额外展示：**

| 字段 | 控件 | 说明 |
|------|------|------|
| 版本升级类型 | Radio Group | patch / minor / major，选择后自动更新版本号建议 |

**版本号校验规则：**
- 必须匹配正则 `/^v\d+\.\d+\.\d+$/`，不符合时阻断提交，提示「版本号格式不正确，需为 vX.Y.Z」
- 版本号为必填，为空时阻断提交

**发布成功后的系统行为：**
1. 生成不可变版本快照，追加到资产的 `history` 事件列表
2. 资产 `savedStatus` 更新为 `'published'`
3. 资产 `currentVersion` 更新为新版本号
4. 显示成功 Toast：「版本 vX.Y.Z 发布成功」
5. 弹窗关闭

#### 4.3.4 回滚确认弹窗（F-04）

**触发：** 点击历史版本卡片「回滚」按钮。

**弹窗结构：**

```
标题：回滚至 v1.0.0 版本

┌──────────────────────────────────────────┐
│  回滚版本 v1.0.0      xxx · 2026-01-20   │
│  版本说明文字内容（若无则显示默认描述）      │
└──────────────────────────────────────────┘

此操作会：
• [模块差异描述] 配置恢复至选中的 v1.0.0 版本状态
• 草稿内容更新为目标版本内容，覆盖当前编辑内容
• 历史记录保留，可随时回滚到其他版本

              [取消]   [确认回滚]（深色主按钮）
```

**回滚成功后的系统行为：**
1. 当前草稿（表单/编辑器内容）替换为目标版本快照数据
2. `savedStatus` 重置为 `'draft'`
3. 历史版本抽屉关闭
4. 显示成功 Toast：「已回滚至 vX.Y.Z」

#### 4.3.5 版本号自动建议逻辑（F-05）

```
规则：基于上次发布版本号 + 升级类型，生成建议值

  patch：v1.1.0  →  v1.1.1   （最小改动，如提示词微调）
  minor：v1.1.0  →  v1.2.0   （中等改动，如新增技能/维度）
  major：v1.1.0  →  v2.0.0   （重大重构）

特殊情况：无历史发布版本时，建议 v1.0.0

用户可在建议值上直接修改，确认时进行格式校验。
```

### 4.7 评测（Evaluation）模块

评测模块含三个子模块，版本控制设计有差异：

| 子模块 | 版本控制 | 权限控制 |
|--------|:--------:|:--------:|
| 评测任务 | ❌ 不引入 | ✅ 创建人限制 + Scope 隔离 |
| 评测集 | ✅ 含数据行快照 | ✅ Scope（发布时选择） |
| 评估器 | ✅ | ✅ Scope（发布时选择） |

#### 4.7.1 评测任务：权限控制设计（F-22 / F-23）

**设计决策：** 评测任务是一次性执行配置，不需要版本历史，但需要访问范围管理与操作归属控制。

**新增 `scope` 字段：**

```typescript
interface EvalTask {
  id:            number;
  name:          string;
  datasetName:   string;
  evaluatorName: string;
  status:        'pending' | 'running' | 'done' | 'failed';
  scope:         'private' | 'team' | 'public';   // 新增
  createdBy:     string;
  createdAt:     string;
  updatedAt:     string;
}
```

**列表页三 Tab 视图（F-22）：**

```
[我的任务（N）]    [团队任务（N）]    [公开任务（N）]
```

| Tab | 数据过滤规则 |
|-----|------------|
| 我的任务 | `createdBy === currentUser`（包含私有任务） |
| 团队任务 | `scope === 'team' \|\| scope === 'public'` |
| 公开任务 | `scope === 'public'` |

**可见范围列（新增表格列）：**

| scope | Tag 颜色 | 图标 |
|-------|---------|------|
| private | default（灰）| `<LockOutlined />` |
| team | blue | `<TeamOutlined />` |
| public | green | `<GlobalOutlined />` |

**操作权限控制（F-23）：**

- 编辑、删除按钮：仅当 `record.createdBy === currentUser` 时为可用状态
- 非创建人时：按钮渲染为 `disabled`，Hover 展示 Tooltip「仅创建人可编辑 / 仅创建人可删除」
- `Popconfirm` 同步设为 `disabled`，防止键盘事件绕过

**新建/编辑表单新增字段：**

```
可见范围  （必填，默认值：team）
  ○  🔒  仅自己  ——  仅你自己可见
  ●  👥  团队内  ——  团队成员可见
  ○  🌐  公开    ——  所有人可见
```

#### 4.7.2 评测集：版本控制设计（F-18 / F-19）

评测集存储的是结构化数据行，其版本快照记录的是**某时刻的全量数据行集合**。

**版本快照字段：**

```typescript
snapshot: DataItem[];   // 全量数据行，支持数据级版本恢复
```

**历史事件类型：**

| 事件类型 | 触发时机 | 记录内容 |
|---------|---------|---------|
| `publish` | 点���发布 | 版本号、变更日志、Scope、全量数据快照 |
| `save` | 点击保存草稿 | 摘要（本次保存了 X 条数据）、全量数据快照 |

**发布弹窗附加版本升级类型（F-19）：**

| 类型 | 含义 | 版本变更 |
|------|------|---------|
| patch | 数据微调 / 少量增删 | x.x.+1 |
| minor | 较大规模数据更新 | x.+1.0 |
| major | 数据集重大重构 | +1.0.0 |

**版本预览态（DatasetDetail 全页）：**
- 顶部黄色 Banner（通用规范）
- 数据表格变为只读：新增行、编辑行、删除行操作均隐藏
- 发布按钮、保存按钮隐藏

#### 4.7.3 评估器：版本控制设计（F-20 / F-21）

**版本快照字段：**

```typescript
snapshot: {
  name: string;         // 评估器名称
  type: 'LLM' | 'Code'; // 评估器类型
  desc: string;         // 描述
}
```

**发布弹窗附加版本升级类型（F-21）：**

| 类型 | 含义 | 版本变更 |
|------|------|---------|
| patch | 评估逻辑微调 | x.x.+1 |
| minor | 新增评估维度 | x.+1.0 |
| major | 重大变更 | +1.0.0 |

**历史版本抽屉：** 附属于列表页，宽度 440px，遵循通用规范（§4.3.1）。

**版本预览快照展示卡片内容：**

```
版本快照
  名称：通用一致性评估
  类型：LLM
  描述：基于LLM对回答与参考结果进行一致性判断
```

### 4.8 异常与边界场景处理

| 场景 | 处理策略 |
|------|---------|
| 资产无任何历史事件 | 不渲染「历史版本」入口按钮，不展示历史版本抽屉 |
| 版本号格式不合法 | 阻断发布表单提交，Toast 提示「版本号格式不正确，需为 vX.Y.Z」 |
| 预览态下触发编辑 | 字段 `disabled` 无法输入，快捷操作按钮已隐藏，从源头阻断 |
| 回滚后草稿未发布离开 | `savedStatus` 保持 `draft`，草稿数据不丢失（离开拦截弹窗列入下期） |
| 首次发布（无历史版本） | 系统建议版本号 `v1.0.0`，用户可修改 |
| 多人同时编辑同一资产 | 本期前端无冲突检测，后端接入后补充乐观锁机制 |
| 非创建人尝试操作评测任务 | 按钮置灰 + Tooltip 提示，Popconfirm 同步禁用，前端拦截 |

### A. 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 草稿 | Draft | 未正式发布的当前编辑状态，对外不可见 |
| 版本快照 | Snapshot | 发布时生成的不可变全量配置存档 |
| 历史事件 | History Event | 每次保存或发布操作在时间线上的记录条目 |
| 回滚 | Rollback | 将草稿内容覆写为指定历史版本快照 |
| 可见范围 | Visibility Scope | 控制资产对哪些用户可见的三级权限设定 |
| 版本预览 | Preview Mode | 只读浏览历史快照的临时状态，不改变草稿 |
| 发布 | Publish | 将当前草稿打包为正式版本，生成快照并记录版本号 |
| 创建人 | Owner | 资产的创建者，拥有完整操作权限 |
