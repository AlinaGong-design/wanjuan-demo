# 产品需求文档（PRD）- 智能体群组协作系统

**文档版本**: v1.0
**创建日期**: 2026-01-26
**产品定位**: AI智能体群组协作与任务编排平台

---

## 一、产品概述

### 1.1 产品简介
智能体群组协作系统是一个类Discord/Slack的AI协作平台，通过群组化管理多个AI智能体，实现复杂任务的自动拆解、分发和协同执行，以群聊形式可视化展示整个工作过程。

### 1.2 核心价值
- **多智能体协作**: 通过群组模式让多个专业智能体协同工作
- **任务自动编排**: 调度智能体自动拆解任务并分配给合适的执行智能体
- **可视化交互**: 以群聊形式实时展示智能体的工作过程和结果
- **灵活配置**: 通过Skill插件系统扩展智能体能力

### 1.3 目标用户
- 企业团队协作用户
- AI应用开发者
- 需要多专业领域AI支持的个人用户

### 【前台】M1: 群聊交互模块

#### 3.1.1 页面布局
**页面路径**: `/` (Frontend.tsx)

**布局结构**:
```
┌─────────────────────────────────────────────┐
│ Header (标题 + 用户信息)                      │
├──────────┬──────────────────────────────────┤
│          │  聊天窗口区域                     │
│  左侧栏  │  ┌──────────────────────────┐    │
│          │  │  消息列表                 │    │
│  - 智能体│  │  - 用户消息               │    │
│  - 群组  │  │  - 智能体回复             │    │
│          │  │  - 任务执行进度           │    │
│          │  └──────────────────────────┘    │
│          │  ┌──────────────────────────┐    │
│          │  │  输入框（富文本）         │    │
│          │  │  - @ 提及按钮             │    │
│          │  │  - 发送按钮               │    │
│          │  └──────────────────────────┘    │
└──────────┴──────────────────────────────────┘
```

#### 3.1.2 核心功能

**功能1: 消息展示**
- 展示用户消息和智能体回复
- 区分不同智能体（通过头像、名称、颜色）
- 支持消息时间戳显示
- 支持富文本内容渲染（包括@提及高亮）

**功能2: 智能体工作状态可视化**
- 显示智能体"正在输入..."状态
- 展示任务拆解后的子任务列表
- 实时更新子任务执行进度（待执行/执行中/已完成）
- 显示每个子任务的负责智能体

**功能3: 消息交互操作**
- 点赞/点踩反馈
- 重试生成
- 引用回复
- 复制消息内容

**功能4: 富文本输入**
- 支持多行文本输入
- @提及智能体/群组
- 快捷键发送（Enter/Ctrl+Enter可配置）
- 输入框自适应高度

#### 3.1.3 业务流程

**流程1: 单智能体对话**
```
用户输入 → 发送消息 → 智能体接收 → 智能体回复 → 展示结果
```

**流程2: 群组协作对话**
```
用户输入 → 发送到群组 → 调度智能体接收
    ↓
任务分析与拆解
    ↓
创建子任务列表 → 分配给成员智能体
    ↓
各智能体并行执行 → 实时更新进度
    ↓
所有子任务完成 → 调度智能体汇总结果 → 展示最终回复
```

## 四、核心业务流程

### 4.1 群组创建流程

```
开始
  ↓
用户点击"新建群组"
  ↓
打开创建弹窗
  ↓
填写群组信息
  ├─ 输入群组名称
  ├─ 选择调度智能体（必选）
  └─ 选择成员智能体（多选）
  ↓
点击"创建"按钮
  ↓
验证表单
  ├─ 名称非空？
  ├─ 是否选择调度智能体？
  └─ 是否至少有1个成员？
  ↓
保存群组数据
  ↓
左侧栏群组列表更新
  ↓
自动打开新建的群组聊天窗口
  ↓
结束
```

### 4.2 群组协作任务流程

```
用户在群组中发送消息
  ↓
消息发送到群组的调度智能体
  ↓
调度智能体分析任务需求
  ↓
判断: 是否需要拆解为子任务？
  │
  ├─ 否 → 调度智能体直接回复 → 结束
  │
  └─ 是 → 继续
        ↓
    任务拆解
      ├─ 子任务1: 分配给智能体A
      ├─ 子任务2: 分配给智能体B
      └─ 子任务3: 分配给智能体C
        ↓
    在聊天窗口展示子任务列表
        ↓
    各智能体并行执行任务
      ├─ 智能体A: 状态"执行中" → 完成 → 状态"已完成"
      ├─ 智能体B: 状态"执行中" → 完成 → 状态"已完成"
      └─ 智能体C: 状态"执行中" → 完成 → 状态"已完成"
        ↓
    实时更新进度条/任务状态
        ↓

        ↓

        ↓
```

```
       ▼

└──────┬───────┘                    └──────┬───────┘
       │                                   │
       │ 1:1                               │ N:M
       │ coordinator                       │
       └───────────────────────────────────┘
                                           │
                                           │ 关联
                                           ▼
                                    ┌──────────────┐
                                    │   Skill      │
                                    │  (技能)      │
                                    └──────────────┘

┌──────────────┐        N:1        ┌──────────────┐
│ ChatMessage  │───────────────────►│   Agent      │
│  (消息)      │   sentBy           │  (智能体)    │

       ▼
```

## 七、交互设计要点

### 7.1 视觉规范

**颜色系统**:
- 智能体提及: 蓝色 (#1677ff)
- 群组提及: 绿色 (#52c41a)
- 调度智能体消息: 特殊背景色
- 子任务状态:
  - 待执行: 灰色 (#d9d9d9)
  - 执行中: 蓝色 (#1677ff) + Loading动画
  - 已完成: 绿色 (#52c41a)

**图标规范**:
- 统一使用emoji作为智能体图标
- 支持自定义品牌色

### 7.2 交互细节

**输入框**:
- 支持Shift+Enter换行
- Enter发送（可配置）
- @触发提及面板
- 输入框自适应高度（最大5行）

**消息展示**:
- 智能体消息带头像和名称
- 支持消息重试
- 支持点赞/点踩反馈

**任务进度**:
- 实时更新子任务状态
- 支持展开/折叠任务详情
- 显示执行时长

## 八、技术架构说明（简略）

- React 18 + TypeScript

- Slate.js（富文本编辑器）

- **实时更新**: WebSocket / Server-Sent Events
